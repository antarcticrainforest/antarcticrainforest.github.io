h1>Privacy Policy — Canon Photo Uploader (Personal)

Effective date: 16 August 2025

Summary

This is a personal desktop script used by the developer to upload locally stored photos to Google Photos. It has no server backend. All processing happens on the user’s own machine.

Data the app accesses

  • Google Photos (app-created data only): The app uploads images and creates/uses an app-created album (“Canon”). It may set the item description. It does not read, list, or modify user-created or shared albums/items outside the app-created scope.
  • Local files: JPEG files selected from the user’s computer are read for upload.
  • OAuth tokens: A Google OAuth refresh token and access tokens are stored locally on the user’s device to enable uploads without repeated sign-in.

How the data is used

  • To upload the selected JPEG files to the user’s Google Photos library.
  • To create/find the “Canon” album and add uploaded items to it.
  • (Optional) To set a textual description on app-created items.

Data sharing

No personal data or media is shared with third parties. The app communicates only with Google’s Photos Library API.

Data storage & security

  • No server-side storage. The app stores a Google OAuth credential file (including a refresh token) only on the user’s device (e.g., ~/.config/photo-uploader/photos_token.json).
  • Users can delete this file at any time and revoke access in their Google Account settings (Security → Third-party access).
  • Network communication to Google uses HTTPS.

User controls

  • Delete or manage uploaded items directly in Google Photos.
  • Revoke the app’s access in Google Account → Security → Third-party access.
  • Delete the local credential file to remove stored tokens on the device.

Legal basis & GDPR (EU/EEA)

Processing is based on the user’s consent (OAuth). As an EU user, you may request access, correction, or erasure of your personal data handled by this app (primarily the locally stored credential file). For Google Photos content itself, use Google Photos’ own data controls.

Children’s privacy

This app is not directed to children under 16 and should not be used by them.

Changes

Any material changes to this policy will be updated on this page with a new effective date.
